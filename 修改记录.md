### 【修改 #7】2026-05-30 —— 血量蓝量系统 + 攻击扣血生效 + 地图怪物全部活化

#### 修改原因
用户反馈三个严重问题：
1. **玩家和怪物缺乏血量蓝量系统**：玩家需要 HP/MP，每秒恢复 1 点；怪物需要有血量条显示

**player.cpp 实现**：

```cpp
void Player::takeDamage(int dmg)
{
    hp -= dmg;
    if (hp < 0) hp = 0;
    qDebug() << "Player took damage:" << dmg << "HP:" << hp << "/" << maxHp;
}

bool Player::consumeMp(int cost)
{
    if (mp < cost) {
        qDebug() << "Not enough MP! Need" << cost << "have" << mp;
        return false;
    }
    mp -= cost;
    qDebug() << "Consumed MP:" << cost << "MP left:" << mp << "/" << maxMp;
    return true;
}

void Player::recoverHpMp(int hpAmount, int mpAmount)
{
    hp += hpAmount;
    if (hp > maxHp) hp = maxHp;
    mp += mpAmount;
    if (mp > maxMp) mp = maxMp;
}
```

**enemy.cpp 血条更新**（takeDamage 中）：

```cpp
void Enemy::takeDamage(int dmg)
{
    if (dead) return;
    hp -= dmg;
    qDebug() << "Enemy took damage:" << dmg << "HP left:" << hp << "/" << maxHp;

    // 更新血条宽度比例
    if (hpBarFg && maxHp > 0) {
        qreal ratio = static_cast<qreal>(hp) / maxHp;
        qreal width = 32.0 * ratio;
        hpBarFg->setRect(-16, -10, width, 4);
    }

    if (hp <= 0) {
        hp = 0;
        dead = true;
        // 死亡时隐藏血条
        if (hpBarBg) hpBarBg->hide();
        if (hpBarFg) hpBarFg->hide();
    }
}
```

> **血条设计**：宽 32px、高 4px，位于怪物头顶上方 10 像素处。背景灰色，前景红色。受伤时前景矩形宽度按比例缩小。

#### 7.5 对后续开发的影响

| 影响点 | 说明 |
|--------|------|
| **玩家死亡处理** | 当前 `Player::takeDamage()` 只扣血不处理死亡。后续 HP <= 0 时可触发死亡动画、回到出生点、游戏结束界面 |
| **Boss 血量** | 当前 Boss 仍是静态 Tile，无血量系统。如需 Boss 战，可将 Boss Tile 替换为 Enemy 子类或独立 Boss 类 |
| **伤害数字飘字** | 当前只有 qDebug 输出伤害。后续可在场景中创建临时 QLabel/QGraphicsTextItem 显示飘字 |
| **技能冷却 + MP 消耗 UI** | 建议在画面上方添加 HUD：玩家 HP/MP 条、技能图标冷却遮罩 |
| **敌人数量优化** | chamber1 有 88 个 Enemy，密集时可能较多炮弹。如需优化，可减少 minion_image 图层密度，或限制同屏敌人最大数量 |
