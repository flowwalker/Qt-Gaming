**player.cpp 实现**：

```cpp
void Player::takeDamage(int dmg)
{
    hp -= dmg;
    if (hp < 0) hp = 0;
    qDebug() << "Player took damage:" << dmg << "HP:" << hp << "/" << maxHp;
}

bool Player::consumeMp(int cost)
{
    if (mp < cost) {
        qDebug() << "Not enough MP! Need" << cost << "have" << mp;
        return false;
    }
    mp -= cost;
    qDebug() << "Consumed MP:" << cost << "MP left:" << mp << "/" << maxMp;
    return true;
}

void Player::recoverHpMp(int hpAmount, int mpAmount)
{
    hp += hpAmount;
    if (hp > maxHp) hp = maxHp;
    mp += mpAmount;
    if (mp > maxMp) mp = maxMp;
}
```

```cpp
void Enemy::takeDamage(int dmg)
{
    if (dead) return;
    hp -= dmg;
    qDebug() << "Enemy took damage:" << dmg << "HP left:" << hp << "/" << maxHp;

    // 更新血条宽度比例
    if (hpBarFg && maxHp > 0) {
        qreal ratio = static_cast<qreal>(hp) / maxHp;
        qreal width = 32.0 * ratio;
        hpBarFg->setRect(-16, -10, width, 4);
    }

    if (hp <= 0) {
        hp = 0;
        dead = true;
        // 死亡时隐藏血条
        if (hpBarBg) hpBarBg->hide();
        if (hpBarFg) hpBarFg->hide();
    }
}
```

```cpp
    case Qt::Key_Equal:

        if (zoomLevel > MAX_ZOOM) zoomLevel = MAX_ZOOM;
```

### 【修改 #12】2026-05-30 —— 保留原始分辨率 + 修复技能中心偏移

#### 修改原因
放大时图片失真：高分辨率素材（player.gif 246×240、wall_2.png 372×346）被提前缩放到 32×32，放大只能拉伸低分辨率图片。

#### 方案
保留原始分辨率，用 `QGraphicsItem::setTransform(QTransform::fromScale())` 缩放到目标显示大小。放大时清晰度显著提升。

#### 修改的文件
| 类型 | 文件 | 说明 |
|------|------|------|
| ✏️ 修改 | `tile.cpp` | 去掉 `QPixmap::scaled()`，改用 `setTransform()` |
| ✏️ 修改 | `player.cpp` | 去掉 `movie->setScaledSize(32,32)`，改用 `setTransform()` |
| ✏️ 修改 | `enemy.cpp` | 去掉直接 `setPixmap()` 后无缩放，改用 `setTransform()` |
| ✏️ 修改 | `game.cpp` | 所有 `player->pos() + boundingRect().center()` 改为 `sceneBoundingRect().center()` |

**tile.cpp 核心改动**：
```cpp
    setPixmap(pixmap);
    setTransformationMode(Qt::SmoothTransformation);
    if (fixedSize.isValid()) {
        qreal sx = static_cast<qreal>(fixedSize.width())  / pixmap.width();
        qreal sy = static_cast<qreal>(fixedSize.height()) / pixmap.height();
        setTransform(QTransform::fromScale(sx, sy));
    }
```

**player.cpp 核心改动**：
```cpp
    movie->start();
    QSize origSize = movie->currentPixmap().size();
    qreal sx = 32.0 / origSize.width();
    qreal sy = 32.0 / origSize.height();
    setTransform(QTransform::fromScale(sx, sy));
```

**game.cpp 技能中心修复（关键 bug）**：
```cpp
// 之前（错误）：boundingRect() 返回未变换的 246×240，与 pos() 不在同一坐标系
QPointF center = player->pos()
                 + QPointF(player->boundingRect().width() / 2.0,
                           player->boundingRect().height() / 2.0);

// 之后（正确）：sceneBoundingRect() 考虑变换，center() 就是真正的视觉中心
QPointF center = player->sceneBoundingRect().center();
```

> **注意**：闪现逻辑中 `player->setPos()` 仍使用左上角（`player->pos()`），轨迹计算单独使用 `oldCenter/finalCenter`，两者不能混用。

| **boundingRect 陷阱** | 任何使用 `pos() + boundingRect().center()` 的代码都会因 transform 而失效，必须用 `sceneBoundingRect().center()` |
| **队友更新地图** | 只要图层名不变（wall、minion_image 等），代码自动适配。新增图层需加入 `targetLayers` 并写处理逻辑 |
