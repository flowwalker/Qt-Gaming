## 一、项目架构总览

```
Qt-Gaming-main
├── main.cpp              → 程序入口，创建 QApplication 和 Game 窗口
├── game.h/cpp            → 游戏主控类 (~4050行)：窗口、键盘输入、游戏循环、地图切换、
│                           传送门、全部技能、敌人管理、HUD、宠物、巢穴、菜单系统、
│                           钻石系统、小地图、梅花瓣/暴风雪粒子、地图介绍、管理员模式、
│                           音频系统(BGM+SFX)、危险区域轰炸、J连击、子空间区域叠加
├── player.h/cpp          → 玩家类 (~323行)：GIF多状态动画、移动、碰撞、HP/MP/EXP、等级进化、
│                           变身形态、施法动画(支持保持帧)、跨地图状态保持、缩小形态
├── tilemap.h/cpp         → 地图管理类 (~174行)：加载瓦片、墙壁/水碰撞（空间索引网格）
├── tile.h/cpp            → 瓦片类 (~25行)：图片显示单元，保留原始分辨率 + transform 缩放
├── maploader.h/cpp       → 地图加载器 (~137行)：解析 Tiled JSON (.tmj) 地图文件
├── skill.h/cpp           → 技能投射物类 (~487行)：火球/简易弹/破空梭/月牙弹/刀浪/GIF刀浪
├── enemy.h/cpp           → 敌人类 (~310行)：随机游走 AI、8种GIF外观随机、三方向炮弹攻击、血条
├── pet.h/cpp             → 宠物类 (~138行)：跟随玩家、Idle/Run 状态切换、11方向DFS绕墙
├── spawner.h/cpp         → 巢穴类 (~91行)：定时生成怪物波次、漩涡外观、血量倍率递增、安全放置算法
├── resources.qrc         → Qt 资源文件：注册 170+ 图片/地图/GIF/音频文件
└── Qt-Gaming.pro         → Qt 项目配置文件：widgets+multimedia模块、console输出
```

### 构建系统

- **构建工具**：qmake（Qt-Gaming.pro）
- **Qt 版本**：Qt 6.11（widgets + multimedia 模块）
- **编译**：`make clean && qmake && make -j$(sysctl -n hw.ncpu)`
- **产物**：`Qt-Gaming.app`
- **注意**：修改 `.qrc` 或 `.pro` 后必须 `make clean`

### 数据流

```
Tiled 地图(.tmj) → MapLoader 解析 JSON → TiledMapData 结构体
                                            ↓
                              Game::loadMap() 二次解析 JSON 手动绘制所有图层
                                            ↓
                              创建 Tile、Enemy、Spawner、Pet、钻石、传送门精灵
                                            ↓
                              Player 接收键盘输入 → move() → 碰撞检测(墙+水)
                                            ↓
                              updateGame() 60FPS 主循环：
                                - 传送过渡(黑幕渐暗→传送→渐亮)
                                - 子空间区域检测 + 树/雾/屋顶透明度
                                - 闪现动画处理(含 rush_jumb GIF)
                                - 定身状态检查
                                - J连击窗口计时 + 连播放映
                                - 突进平滑移动
                                - 加速冷却
                                - 玩家移动 + 水碰撞回退
                                - HP/MP 恢复 (每20帧)
                                - 投射物更新 + 碰撞伤害
                                - 玄武盾位置更新
                                - 地形效果 (火焰扣血)
                                - 宠物更新
                                - 背后火焰动画
                                - 敌人 AI + 攻击 + 死亡奖励
                                - 敌人炮弹更新 + 玩家受伤
                                - 巢穴计时生成
                                - HUD 更新
                                - 传送门旋转 + 检测
                                - 特殊坐标传送检测 (161,40↔149,40)
                                - 宝箱/门交互
                                - 地图介绍检测
                                - 雾透明度渐变
                                - 小地图更新
                                - 钻石动画/碰撞
                                - 梅花瓣/暴风雪粒子
                                - 危险区域更新(lianda)
```


##### loadMap() 函数 — 最核心函数

完整流程总结（关键新增标注 ★）：

```
1. 停止 gameTimer
2. 清理旧资源（技能栏、危险区域、tileMap）
3. 保存跨地图状态
4. 删除旧 player
5. 清理投射物(6种)、盾牌、敌人/炮弹、巢穴、宠物
6. 清理变身动画QMovie（先stop再delete防野指针）
7. 清理背后火焰、HUD(全部10个)、小地图
8. 清理钻石列表、传送门精灵、buffIndicator
9. 清空 hittableItems/chests/doors/fireRects/portal过渡状态
10. 清空场景所有项
11. 创建新 TileMap → loadFromFile()
```

## 九、西南联大危险区域系统

### 9.1 机制

- **仅 lianda 地图**触发（`currentMapPath.contains("lianda")`）
- 每 **180帧（3秒）** 在玩家当前位置生成一个 DangerZone
- DangerZone 持续 **150帧（2.5秒）**，缓慢随机漂移
- 倒计时结束 → 爆炸（`createExplosion(pos, 480)`）+ 范围扣血

### 9.2 DangerZone 结构

```
DangerZone:
  ├── circle: QGraphicsEllipseItem — 红色圆圈(半径240px=480px直径，Z=50)
  ├── cross1/cross2: QGraphicsLineItem — 红色叉号 × (Z=51)
  ├── pos: QPointF — 生成位置
  ├── driftVel: QPointF — 随机漂移速度(0.5px/帧)
  └── lifetime: 150帧 (2.5秒)
```

### 9.3 伤害判定

- 爆炸时：玩家 hitboxRect 与 480×480 矩形相交 → 扣 20% 最大HP
- 爆炸音效：`explosion-01.mp3`(0.3)（在 createExplosion 内触发）
- 出现警告音效：`mixkit-arcade-rising-231.wav`(0.3)
